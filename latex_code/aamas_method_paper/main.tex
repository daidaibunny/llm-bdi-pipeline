\begin{abstract}
We study how to turn generalized-planning artifacts into maintained
AgentSpeak(L) plan libraries. The method separates two tasks that are often
conflated. First, an external generalized planner learns lifted templates for
atomic predicate or literal goals from a Planning Domain Definition Language
domain and its training split. Second, each user temporal goal is represented as
a lifted LTLf formula, compiled into a deterministic finite automaton, validated
under a conjunction-and-negation guard contract, and appended as a query-specific
top-level AgentSpeak(L) goal that calls the atomic library. The resulting
artifact is one domain-level library that grows with user queries, rather than
one plan per problem or one planner call per execution. An Evidence Module
imports external generalized-planning artifacts into a provider-neutral
singleton-goal evidence representation. We instantiate this interface with
MOOSE, while keeping the compiler independent of that provider. A single
Clingo program jointly selects validated evidence macros and schema-derived
branches under executability, closure, recursive-progress, resource-restoration,
and target-preservation certificates. Acyclic producer dependencies through
schema-inferred single-valued fluents permit certified existential context
projection, while cyclic or ambiguous
dependencies retain their full preparation context. Conditional completion effects certify
both positive-sibling preservation and negative-guard non-addition. Certified conjunctive DFA guards are
executed through balanced query-local repair trees that bound AgentSpeak plan
fan-out while preserving the certified literal order. Unsupported candidates
and uncertified cyclic transition guards are rejected before library generation.
\end{abstract}

\section{Introduction}

Classical planners can generate a plan for a concrete problem, but a
Belief-Desire-Intention agent requires a reusable plan library. AgentSpeak(L)
provides a logical language for such agents \cite{Rao1996AgentSpeak}. This
paper targets the gap between generalized planning and executable agent
programs: given a \pddl{} domain and a small set of training problems, produce a
lifted \asl{} library whose atomic predicate-goal templates can be reused by
future user queries. The resulting artifact is ordinary agent code: a finite set
of lifted plan schemas whose triggers, contexts, and bodies can be inspected
independently of any particular test instance.

The central difficulty is the interface between reusable atomic behavior and
query-specific temporal structure. The Evidence Module normalizes provider
artifacts before compilation. Our MOOSE instantiation uses goal-regression
rules that decompose training problems into singleton goal conditions
\cite{Chen2026Moose}. This makes MOOSE a natural evidence provider for atomic
templates such as \texttt{on(X,Y)} or \texttt{served(P)}, but not a complete
answer for interacting multi-literal goals. In Blocksworld, for example,
\texttt{on(X,Y)} can be a useful atomic template, while a tower request such as
\texttt{on(X,Y)} followed by \texttt{on(Y,Z)} needs explicit temporal or
structural control. We place that control in a query-specific LTLf-to-DFA layer
above the atomic library.

The paper makes five contributions.
\begin{enumerate}
\item We define an architecture that separates atomic lifted template learning
	from query-specific temporal goal appending.
\item We define an Evidence Module interface for atomic predicate and literal
	templates. The MOOSE adapter is one provider that emits the common
	singleton-goal evidence representation consumed by the compiler.
\item We define one certificate-carrying policy-lifting compiler that jointly
	selects evidence and schema-derived atomic modules in a single Clingo
	optimization problem, including acyclic existential preparation projection
	with schema-derived feasibility witnesses.
\item We restore an LTLf-to-DFA temporal layer whose conjunctive guards use
	conservative conditional add/delete summaries, enforced negative-guard
	preservation, and balanced repair trees with bounded trigger fan-out.
\item We define a benchmark scope spanning classical, numeric, and
	serialized-width domains.
\end{enumerate}

The contribution is deliberately bounded. We do not claim that one evidence
provider is state of the art for every planning problem, nor that context-only
\asl{} can encode every temporal automaton without controller state.
Unsupported negative achievement templates, disjunctive guards, cyclic
delete-threat graphs without a certified ranking or the stated acyclic-support
and branch-noninterference assumptions, safety-sensitive formulas requiring
primitive-step monitoring, and multi-literal numeric
guards without numeric preservation certificates remain outside the claim.

\section{Current Scope and Exclusions}
\label{sec:scope}

The current method supports PDDL predicate goals and validated numeric fluent
goals with lifted atomic templates. Negated predicates are supported as
completion-level context guards only when conditional may-add preservation is
certified; negative achievement templates remain unsupported. Disjunctive
goals, derived predicates, and conditional effects require additional semantics
before they can enter the claim. Unsupported cases are
rejected before compilation so that the generated library remains a
domain-level program rather than an instance-specific trace.

Temporal goals are compiled from LTLf into deterministic finite automata. DFA
transition guards may contain conjunction and negation. Positive conjuncts call
atomic modules; negative conjuncts remain certified context guards and are never
invented as negative achievement calls. A general DFA still requires an external
reward-machine state unless the world state uniquely determines temporal
progress. Ordinary classical \pddl{} goals define achievement goals; they do
not define arbitrary temporal extended goals, although PDDL3 trajectory
constraints can express part of that space.
