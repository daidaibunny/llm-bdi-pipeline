\begin{abstract}
Generalized planning synthesizes reusable behavior for families of planning
problems, whereas Belief--Desire--Intention (BDI) agents execute plan libraries.
The two representations are not interchangeable: a learned policy need not
provide executable plans for internally required subgoals or support safe
composition of temporally extended requirements. We introduce \methodname{}
(Generalized Planning to Plan Libraries), which closes this representation gap
by compiling generalized-planning evidence and Planning Domain Definition
Language (PDDL) action models into certified BDI plan libraries. GP2PL validates
and lifts evidence rules, completes missing achievable subgoals from action
schemas, selects a compact executable atomic core, and appends query-specific
controllers for finite-trace temporal goals without relearning that core. We
formalize this representation, selection, and composition process and prove
soundness for the supported fragment; unsupported obligations are rejected.
Across \BenchmarkDomainCount{} domains, five independently seeded libraries
solve \AtomicFiveSeedMeanSuccessPercent{}\% of held-out cases on average
(\AtomicFiveSeedSampleSDPercent{} percentage-point sample standard deviation).
All \TranslationEquivalentCount{} translated temporal specifications match
their gold finite-trace languages, and all \TEGProblemCount{} bound queries
produce independently validated accepting traces.
\end{abstract}

\section{Introduction}

Classical PDDL planning computes an action sequence for one grounded problem
\cite{McDermott1998PDDL}. Generalized planning instead learns reusable behavior
for a family of related problems
\cite{Srivastava2011DirectedSearch,Chen2026Moose}. A
Belief--Desire--Intention (BDI) interpreter uses a different representation: a
plan library that connects events and belief contexts to actions and subsidiary
goals \cite{Rao1996AgentSpeak,MeneguzziDeSilva2015PlanningBDI}. Turning a
generalized policy into such a library is therefore a compilation problem, not
a change of syntax.

Blocks World is a standard planning domain in which a robot rearranges blocks.
A learned rule for placing block $X$ on block $Y$ may
use the action \texttt{stack(X,Y)} when the robot is holding $X$ and the
destination block is clear. If $Y$ is not clear, however, an executable BDI
library also needs a reusable plan for making it clear, even when that condition
never appeared as a training goal. More generally, variables must remain safely
bound, subsidiary goals must resolve to library plans, and recursive plans must
make progress.

Temporally extended goals add a second representation gap
\cite{DeGiacomoVardi2013,DeGiacomoRubin2018AutomataFOND}. A request to place one
block on another and only later clear a third block constrains a finite state
trace, not only its final state. Likewise, a conjunctive milestone requires its
conditions to hold together, so a later plan must not undo an earlier
achievement. This is a form of BDI goal interference
\cite{Thangarajah2003GoalInterference}. Safe reuse therefore requires temporal
control over the fixed domain library and observation after primitive actions.

Figure~\ref{fig:architecture} separates one-time domain compilation from
query-specific additions. After the atomic core is built, each new temporal
request adds only its controller plans to the same maintained library. Both
stages use symbolic action schemas and effects rather than domain-specific
names.

Figure~\ref{fig:policy-lifting-example} shows how evidence for one block-placement
goal becomes a small, internally closed set of BDI modules. The method sections
define the exact candidate language, certificates, and selection problem.

GP2PL makes three contributions. First, it turns singleton-goal policy evidence
and PDDL action models into a compact BDI module core, including plans for
achievable subsidiary goals omitted by the evidence. Second, it certifies safe
variable binding, executable plan bodies, achieved subgoals, and progress of
recursive plans; otherwise, compilation rejects the library. Third, it appends
controllers that preserve earlier milestones while pursuing supported
finite-trace temporal goals, retaining the same domain core. Across
\BenchmarkDomainCount{} domains, independently compiled atomic cores average
\AtomicFiveSeedMeanSuccessPercent{}\% held-out execution coverage over five
goal-order seeds, and all \TEGProblemCount{} predeclared temporal queries pass
independent execution and trace validation. These are guarantees and results
for the declared fragment and evaluated inputs, not completeness for arbitrary
PDDL--\ltlf{} products; formal definitions, proofs, and the reproducibility
protocol appear in the Technical Supplement, Secs.~2 and~5.

\section{Conclusion}

GP2PL compiles generalized-planning evidence into a certified atomic module core
and appends finite-trace query plans to the unique maintained BDI domain library.
Five-seed atomic coverage averages
\AtomicFiveSeedMeanSuccessPercent{}\%; the predeclared temporal study solves
\TEGProblemCount{}/\TEGProblemCount{}. Guarantees remain conditional on the
candidate-construction grammar, generated certified candidate set, and declared
\ltlf{} fragment.
