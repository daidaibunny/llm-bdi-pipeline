\begin{abstract}
Generalized planners synthesize reusable policies, but a
Belief--Desire--Intention (BDI) interpreter requires an executable plan library
whose modules can be maintained, selected, and called by later intentions.
Direct translation does not establish safe variable binding, internal-module
closure, recursive progress, resource restoration, or preservation among
interacting goals. We present a certificate-carrying compiler that bridges
these representations. Its domain stage normalizes singleton-goal planning
evidence, instantiated here by MOOSE, combines validated evidence macros with
PDDL-schema candidates, and uses Clingo to select a compact lifted AgentSpeak
library under explicit correctness obligations. Its query stage keeps
temporally extended goals separate from reusable domain behavior: a real
\ltlf-to-DFA toolchain produces transition guards, whose positive achievements
and negative obligations are serialized only when certified completion effects
establish preservation. The resulting query-local controllers use balanced
declarative repair trees and execute over the same maintained domain library.
The implementation is fail-closed: unsupported negative achievements,
ambiguous automata, uncertified recursive interference, and mixed numeric
conjunctions are rejected rather than compiled heuristically.
\end{abstract}

\section{Introduction}

A classical planner returns a plan for one grounded problem. A generalized
planner instead seeks a policy or program reusable across a family of problems.
A practical Belief--Desire--Intention (BDI) agent needs a third artifact: a
maintained plan library with lifted event triggers, belief contexts, primitive
actions, and calls between achievement modules. Converting a generalized
policy into such a library is not a formatting step. Variables must remain
safely bound, called modules must be present, recursive branches must terminate,
and an apparently shorter branch must not silently lose behavior evidenced by
the source policy.

Consider a generalized-planning rule for the singleton goal
\texttt{on(X,Y)}. A finite evidence macro may apply \texttt{stack(X,Y)} when
\texttt{holding(X)} and \texttt{clear(Y)} hold, but a reusable BDI module must
also handle states in which \texttt{Y} is obstructed. The desired lifted
structure can call an internal \texttt{!clear(Y)} module and retry
\texttt{!on(X,Y)}. The helper may be absent from the evidence because
\texttt{clear(X)} was never a training goal; it must instead enter through
PDDL add-effect and precondition closure. This example exposes the central
representation gap: evidence for grounded executions does not by itself define
a closed, terminating AgentSpeak library.

Temporally extended goals (TEGs) introduce a separate difficulty: their truth
depends on a finite state trace rather than only its final state. A domain-level
module such as \texttt{+!on(X,Y)} should remain reusable, while a query such as
$F(\mathit{on}(X,Y)\land X(F(\mathit{clear}(Z))))$ requires query-specific
control. Treating a conjunction on one automaton transition as an unchecked
sequence of achievement calls is unsound when a later call can delete an
earlier result. Conversely, embedding each query into the static domain
library would conflate reusable behavior with temporal intent.

We address both gaps with the two-stage architecture in
Figure~\ref{fig:architecture}. The first stage turns singleton-goal
generalized-planning evidence and PDDL schemas into one lifted atomic-module
library. The second compiles supported deterministic-finite-automaton (DFA)
transition guards into query-local controllers over that library. MOOSE
supplies the evidence in our implementation, but its rules are evidence rather
than finished AgentSpeak code. After normalization, acceptance depends on rule
structure, PDDL schemas, and compiler certificates, never on domain or
predicate names.

The implemented contributions are:
\begin{itemize}
\item a certificate-carrying plan-library compiler that jointly selects
validated singleton-goal evidence and PDDL-schema modules in one Clingo
optimization;
\item domain-independent obligations for lifted binding, local execution,
closure, recursive relational progress, one-step resource release, and staged
preparation; and
\item a real \ltlf-to-DFA query compiler that uses conditional completion
effects to preserve positive and negative transition obligations, emitting
balanced declarative BDI controllers without domain-level DFA-state fluents.
\end{itemize}

The claim is intentionally narrower than universal generalized planning or full
temporal synthesis. We prove properties only relative to the generated
candidate space and the supported DFA fragment, and we report rejection when a
required certificate cannot be constructed.

\section{Limitations and Scope}
\label{sec:limitations}

The implemented evidence provider is MOOSE; provider neutrality is an interface
property, not an empirical claim about other backends. Schema-derived action
sequences use a fixed grammar with at most five primitive actions, and resource
cleanup contains at most one certified release action. The compiler does not
claim semantic equivalence between an arbitrary long evidence macro and a
short recursive program.

At the query layer, transition guards may use conjunction and literal negation.
Negative literals are context obligations, not negative achievement calls.
Mixed or multi-literal numeric guards remain unsupported because the compiler
does not yet construct a cross-literal numeric preservation certificate.
Branching automata whose progress cannot be inferred from world state require
an external DFA-state monitor. Formulas that constrain every primitive state,
such as safety properties, also require such a monitor because the current
effect boundary is successful atomic-module completion.

\section{Conclusion}

This work closes a representation gap between generalized planning and BDI
execution. The compiler turns singleton-goal policy evidence and PDDL schemas
into one maintained, lifted AgentSpeak domain library, while a separate
\ltlf-to-DFA stage appends certified query-local controllers for supported
temporally extended goals. The output can be executed by an unmodified Jason
interpreter and independently checked through PDDL and finite-trace validation.
The contribution is therefore not a new generalized planner or BDI semantics,
but a fail-closed compilation problem that makes reusable planning evidence
deployable as explicit BDI behavior.
