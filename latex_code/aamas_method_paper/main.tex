\begin{abstract}
Generalized planning learns reusable behavior for problem families, whereas
Belief--Desire--Intention (BDI) agents execute plan libraries. Direct adaptation
leaves a representation gap: singleton-goal evidence may omit modules for
subsidiary achievements and does not certify temporal composition. We introduce
\methodname{} (Generalized Planning to Plan Libraries), which compiles such
evidence and Planning Domain Definition Language (PDDL) action models into
certified BDI plan libraries. GP2PL validates and lifts evidence, generates
schema-derived modules for producible goals, selects a compact closed
atomic core, and appends query controllers guided by deterministic finite
automata for finite-trace temporal goals without relearning that core. For the
generated candidate language and supported fragment, accepted branches and
controllers satisfy conditional soundness guarantees; uncertified obligations
are rejected. Across \BenchmarkDomainCount{} domains, five independently seeded
cores achieve \AtomicFiveSeedMeanSuccessPercent{}\% mean held-out coverage
($\AtomicFiveSeedSampleSDPercent{}$ percentage-point sample standard deviation).
Recursive candidates improve paired atomic success by 10.42 percentage points,
and preservation certification improves paired temporal success by 9.28
percentage points. All \TranslationEquivalentCount{} translated specifications
match their gold finite-trace languages, and the fixed-core temporal study
produces independently validated accepting traces for all \TEGProblemCount{}
bound queries.
\end{abstract}

\section{Introduction}

Classical PDDL planning computes an action sequence for one grounded problem
\cite{McDermott1998PDDL}. Generalized planning instead learns reusable behavior
for a family of related problems
\cite{Srivastava2011DirectedSearch,Chen2026Moose}. A
Belief--Desire--Intention (BDI) interpreter uses a different representation: a
plan library that connects events and belief contexts to actions and subsidiary
goals \cite{Rao1996AgentSpeak,MeneguzziDeSilva2015PlanningBDI}. Turning a
generalized policy into such a library is therefore a compilation problem, not
a change of syntax.

Blocks World is a standard planning domain in which a robot rearranges blocks.
A lifted achievement $\texttt{on(X,Y)}$ asks that block $X$ rest on block $Y$.
A learned rule for achieving it may
use the action \texttt{stack(X,Y)} when the robot is holding $X$ and the
destination block is clear. If $Y$ is not clear, however, an executable BDI
library also needs a reusable plan for making it clear, even when that condition
never appeared as a training goal. More generally, variables must remain safely
bound, subsidiary goals must resolve to library plans, and recursive plans must
make progress.

Temporally extended goals add a second representation gap
\cite{DeGiacomoVardi2013,DeGiacomoRubin2018AutomataFOND}. A request to place one
block on another and only later clear a third block constrains a finite state
trace, not only its final state. Likewise, a conjunctive milestone requires its
conditions to hold together, so a later plan must not undo an earlier
achievement. This is a form of BDI goal interference
\cite{Thangarajah2003GoalInterference}. Safe reuse therefore requires temporal
control over the fixed domain library and observation after primitive actions.

The central design choice is reuse: temporal requests extend one maintained
library instead of triggering a new domain-level learning run. Both compiler
stages reason over typed action schemas and effects rather than domain names.

\begin{samepage}
GP2PL makes three contributions. First, it compiles singleton-goal evidence and
PDDL schemas into an internally closed BDI core with modules for omitted
producible subgoals.

Second, it accepts only branches with certificates for binding, symbolic
execution, achievement, completion effects, and progress.

Third, it appends preservation-safe finite-trace controllers over the fixed
core and monitors DFA progress after each primitive action.
\end{samepage}

The experiments separate component effects from complete-system behavior.
Across \BenchmarkDomainCount{} domains, independently compiled atomic cores
average \AtomicFiveSeedMeanSuccessPercent{}\% held-out coverage over five
evidence seeds. A fixed-core study validates all \TEGProblemCount{} registered
temporal queries. These claims concern the declared candidate and formula
fragments, not complete planning for arbitrary PDDL--\ltlf{} products. Formal
definitions and proofs appear in Technical Supplement, Secs.~1--2; Sec.~5 gives
the reporting and reproducibility protocol.

\section{Limitations}

MOOSE is the only experimentally instantiated evidence provider. The provider
interface is general, but candidate generation is incomplete: the
candidate-construction grammar admits schema regression only over an acyclic
producer--precondition dependency graph, and recursive or resource behavior
only when the implemented ranking or keyed-mode certificates apply.
Internal-call closure proves type-compatible resolution of
each selected call; it does not guarantee an applicable branch in every
reachable state.

The temporal benchmark contains witness-backed short-horizon queries mined from
rollouts of at most three actions. Its controlled utterances evaluate structured
translation under a fixed vocabulary, not robustness to unrestricted natural
language or arbitrary long-horizon temporal planning. The end-to-end temporal
matrix also conditions on a fixed seed-0 atomic core, so it supports no
cross-seed temporal-robustness claim.

Primitive-step monitoring gives exact trace observation for the declared
formula syntax, but GP2PL does not provide complete action-strategy synthesis.
Unsupported arithmetic, uncertified interference cycles, and progress guards
without an applicable certified repair are rejected. The empirical results
therefore do not imply arbitrary PDDL--\ltlf{} strategy synthesis.

The input boundary nevertheless offers a practical route to future domains.
When an established benchmark supplies a parameterized PDDL problem generator,
as planning competitions have done \cite{Bacchus2001AIPS00}, its revision,
parameters, and random seed can define a reproducible training curriculum.

After reserving a disjoint held-out split and validating every generated
problem against $D$, a compatible generalized-planning backend can learn
evidence from $\mathcal I_{\mathrm{train}}$ for the unchanged GP2PL compiler.
This scales evidence acquisition and domain onboarding, not the supported
PDDL--\ltlf{} fragment or representative-coverage guarantees.

\section{Conclusion}

GP2PL closes the representation gap by converting generalized-planning evidence
and PDDL action schemas into a reusable atomic BDI core, then appending
query-local temporal controllers to the same maintained library.

The output is executable in the evaluated AgentSpeak(L)/Jason realization
because each accepted branch has certified binding, symbolic PDDL
executability, target achievement, and progress, while selected internal calls
resolve within the library. Unproved obligations fail compilation rather than
becoming unchecked plans.

The certified temporal scope consists of bound formulas in the declared $F$,
$X$, strong-$U$, conjunction, and literal-negation syntax whose DFA progress
obligations admit preservation-safe repairs. Within that scope, the measured
component gains and \TEGProblemCount{}/\TEGProblemCount{} fixed-core temporal
result show that certification changes both coverage and controller structure,
while the limitations above delimit what those results establish.
