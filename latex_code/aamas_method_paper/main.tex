\begin{abstract}
Generalized planners synthesize reusable policies, whereas a
Belief--Desire--Intention agent requires an executable plan library whose
modules can be maintained and called by later goals. We present a
certificate-carrying compiler that bridges these representations. A provider
adapter first normalizes singleton-goal generalized-planning rules; our current
implementation instantiates this interface with MOOSE. The compiler combines
validated evidence macros with candidates derived from PDDL action schemas and
uses Clingo to select a compact set of lifted AgentSpeak plans under explicit
binding, executability, achievement, closure, recursive-progress, and
resource-restoration constraints. Query-specific temporal goals are kept
separate from this domain-level library. A real \ltlf-to-DFA toolchain converts
each supported query into transition guards, and the compiler serializes
conjunctive achievements only when conditional completion effects certify
positive-goal and negative-guard preservation. Balanced query-local repair
trees execute the resulting order with trigger fan-out at most two. The method
is deliberately fail-closed: unsupported negative achievements, disjunctive or
ambiguous automata, uncertified recursive interference, and mixed numeric
conjunctions are rejected instead of being compiled heuristically.
\end{abstract}

\section{Introduction}

A classical planner returns a plan for one grounded problem. A generalized
planner instead seeks a policy or program reusable across a family of problems.
A practical Belief--Desire--Intention agent needs a third artifact: a maintained
plan library with lifted event triggers, belief contexts, primitive actions,
and calls between achievement modules. Converting a generalized policy into
such a library is not a formatting step. Variables must remain safely bound,
called modules must be present, recursive branches must terminate, and an
apparently shorter branch must not silently lose the behavior evidenced by the
source policy.

Temporal user goals introduce a separate difficulty. A domain-level module such
as \texttt{+!on(X,Y)} should be reusable, while a query requiring several facts
in one state or across time needs query-specific control. Treating a conjunction
as a sequence of independent achievement calls is unsound when a later call can
delete an earlier result. Conversely, embedding each query into the static
library would destroy the distinction between reusable domain behavior and
temporal intent.

We address these problems with two explicit compilation stages. The first turns
singleton-goal generalized-planning evidence and PDDL schemas into one lifted
atomic-module library. The second compiles supported DFA transition guards into
query-local controllers over that library. MOOSE supplies the evidence in our
implementation, but MOOSE rules are treated as evidence rather than as the
finished AgentSpeak program. All acceptance decisions after normalization are
made from rule structure, PDDL schemas, and compiler certificates, not from
domain or predicate names.

The contributions implemented in this work are:
\begin{itemize}
\item a normalized singleton-goal evidence contract and a certificate-carrying
compiler that jointly selects validated provider macros and schema-derived
AgentSpeak modules in one Clingo optimization;
\item domain-independent certificates for lifted binding, local schema
execution, recursive relational progress, one-step resource release, and
staged preparation through schema-inferred single-valued fluents; and
\item a real \ltlf-to-DFA query compiler that derives conditional completion
effects, rejects uncertified interference, and emits balanced declarative
repair trees without adding DFA-state fluents to the domain library.
\end{itemize}

The claim is intentionally narrower than universal generalized planning or full
temporal synthesis. We prove properties only relative to the generated
candidate space and the supported DFA fragment, and we report rejection when a
required certificate cannot be constructed.

\section{Limitations and Scope}
\label{sec:limitations}

The implemented evidence provider is MOOSE; provider neutrality is an interface
property, not an empirical claim about other backends. Schema-derived action
sequences use a fixed grammar with at most five primitive actions, and resource
cleanup contains at most one certified release action. The compiler does not
claim semantic equivalence between an arbitrary long evidence macro and a
short recursive program.

At the query layer, transition guards may use conjunction and literal negation.
Negative literals are context obligations, not negative achievement calls.
Mixed or multi-literal numeric guards remain unsupported because the compiler
does not yet construct a cross-literal numeric preservation certificate.
Branching automata whose progress cannot be inferred from world state require
an external DFA-state monitor. Formulas that constrain every primitive state,
such as safety properties, also require such a monitor because the current
effect boundary is successful atomic-module completion.
