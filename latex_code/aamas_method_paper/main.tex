\begin{abstract}
Generalized planning learns reusable behavior for problem families, whereas
Belief--Desire--Intention (BDI) agents execute plan libraries. Connecting
generalized planning to BDI execution exposes a representation gap:
singleton-goal evidence may omit modules for subsidiary achievements and does
not establish sound temporal composition. We
introduce
\methodname{} (Generalized Planning to Plan Libraries). Its typed input
contract maps controlled natural-language requests to bound
\ltlf{} query specifications consistent with the planning domain. Its compiler
turns generalized-planning evidence and
PDDL action models into certified BDI plan libraries. GP2PL validates and lifts
evidence, generates schema-derived modules for producible goals, selects a
compact closed atomic core, and appends DFA-guided query controllers without
relearning that core. Within the supported candidate language and query fragment,
GP2PL accepts only branches and controllers satisfying its soundness conditions.
Atomic coverage averages 98.7\% across five seeds; all 475 translations are
DFA-language equivalent to their gold formulas, and all 6,140 admitted temporal
executions pass validation.
\end{abstract}

\section{Introduction}

Classical planning computes an action sequence for one grounded problem
\cite{McDermott1998PDDL}; generalized planning amortizes this effort by learning
reusable behavior for a family of related problems
\cite{Srivastava2011DirectedSearch,Chen2026Moose}. Belief--Desire--Intention
(BDI) agents encode reusable behavior in plan libraries that connect events and
belief contexts to actions and subsidiary goals
\cite{Rao1996AgentSpeak,MeneguzziDeSilva2015PlanningBDI}. Traditionally,
developers supply these libraries by encoding domain-specific behavior and
anticipating applicable situations before deployment, creating a substantial
knowledge-engineering burden
\cite{Meneguzzi2004PropositionalPlanning,MeneguzziDeSilva2015PlanningBDI}.
Generalized-planning evidence offers a basis for automating this construction.
Given a planning domain $D$ and training instances
$\mathcal I_{\mathrm{train}}$, an external generalized-planning backend produces
raw singleton-goal policy evidence $\mathcal E_{\mathrm{raw}}$. This evidence
records how individual achievements can be reached under particular conditions,
but need not provide the callable, context-indexed subsidiary modules or
composition guarantees required by an executable BDI library.

GP2PL addresses this representation gap as shown in
Figure~\ref{fig:architecture}. It compiles the evidence once into a certified,
reusable atomic module core, then reuses that core when compiling each bound
query into the same maintained library. A query may be an achievement
condition or a temporally extended goal (TEG) \cite{Hindriks2009TemporalGoals}.
A bound query assigns each parameter a type-compatible domain object at invocation;
no quantification is implied. The backend is therefore neither replaced nor
rerun for each query.

GP2PL makes four contributions: an evidence-to-library compiler
that adds modules for producible targets omitted from the evidence and selects a
compact, internally closed atomic core; machine-checkable conditions for module
executability, termination, and composability; a typed input contract mapping
controlled natural-language requests to bound
achievement and TEG specifications, evaluated by exact finite-trace language
equivalence on a TEG benchmark; and query-specific
temporal controllers that reuse and extend one maintained BDI library.

\section{Conclusion and Future Work}

GP2PL translates controlled natural-language requests with typed parameters into
parametric \ltlf{} query specifications and compiles generalized-planning
evidence with PDDL schemas into a closed atomic core. Certified DFA controllers
then extend the same maintained BDI plan library for achievement and temporally
extended goals without relearning the core.
Future work should study unrestricted PDDL--\ltlf{} strategy synthesis, richer
PDDL models with numeric, conditional, and durative effects, and scalable
certificate discovery for larger candidate spaces.
For a new domain supplied without instances, parameterized PDDL problem
generators can synthesize a reproducible training curriculum for evidence
learning \cite{Bacchus2001AIPS00}.
