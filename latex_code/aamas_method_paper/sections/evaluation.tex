\section{Experimental Evaluation}
\label{sec:evaluation}

\subsection{Benchmarks and Protocol}

\paragraph{Corpus and repetitions.}
The corpus contains eight classical and four bounded-integer MOOSE domains with
official splits \cite{Chen2026Moose,DillonZChenMooseDataset} and four
predeclared serialized-width families \cite{LipovetzkyGeffner2012Width}:
Blocksworld Clear and On
\cite{Frances2021LearningGeneralPolicies,BonetLearnerPoliciesRepository},
Tower \cite{Bacchus2001AIPS00,PotasscoPDDLInstances}, and Depots
\cite{Bonet2018Features,RLeapD2LRepository}. MOOSE readable policies provide
$\mathcal E_{\mathrm{raw}}$. Following MOOSE, each of five fixed seeds uses every
training instance and three goal orders to compile one core per domain (80
total).

\paragraph{Temporal benchmark construction.}
For each held-out problem, construction discards the original goal and uses
predicate events and bounded-integer changes to instantiate five predeclared
profiles: $F(A\land B)$, $F(A\land\neg B)$,
$F(A\land X(F(B)))$,
$F(A\land X(F(B\land X(F(C)))))$, and strong $A\,U\,B$.
Each selected hidden formula is lifted and rendered deterministically as
controlled English. GPT-5.5 \cite{OpenAI2026GPT55SystemCard} translates 475
deduplicated inputs, which expand to 1,228 witness-backed bound queries.

\paragraph{Controlled inputs and validation.}
Atomic variants share each seed's evidence; temporal variants share the core,
binding, and DFA. Atomic success requires Jason completion and original-goal
VAL \cite{Bordini2007Jason,Howey2004VAL}; translation requires exact predicted--gold
DFA language equivalence; temporal success requires compilation, Jason,
neutral-goal VAL, and acceptance by both DFAs. Every failure remains in the
denominator. Comparisons are paired by held-out identifier, averaging seeded
atomic outcomes within identifiers before inference. Construction details,
limits, costs, and inference rules appear in the Technical Supplement,
Secs.~4--5.

\subsection{Full-System Coverage and Robustness}
\label{sec:full-system-results}

\paragraph{Five-seed atomic robustness.}

Across 80 domain--seed cores, mean held-out coverage is
\AtomicFiveSeedMeanSuccessPercent{}\% $\pm$

\paragraph{End-to-end temporal validation.}
\label{sec:end-to-end-results}
To isolate temporal compilation, the paired study conditions every controller
variant on the same seed-0 core per domain; it does not estimate cross-seed
temporal variability. Its five structures instantiate
$\Phi_{\mathrm{bench}}$, not all of
$\Phi_{\mathrm{syn}}$, and execution includes only queries admitted to
$\Phi_{\mathrm{cert}}(D,\mathcal M_D)$. All
\TranslationEquivalentCount{}/\TranslationInputCount{} are DFA-language
equivalent, and all \TEGProblemCount{} admitted queries pass Jason, neutral-goal
VAL, and both DFA oracles. Median runtime is \TEGMedianRuntimeSeconds{} seconds
(IQR 4.28--8.49), and median trace length is \TEGMedianActionCount{} actions
(IQR 1--2); trace length is descriptive, not an action-cost optimality claim.
Across 740 same-seed added-family cases, GP2PL raises VAL-valid coverage from
15.8\% to 97.3\%; original-scope MOOSE results are separate
\cite{Chen2025MooseExtended}. On the \ExternalDirectSupportedCount{} Boolean
queries in the direct FOND4LTLf comparison, GP2PL validates
\ExternalDirectSupportedCount{}/\ExternalDirectSupportedCount{} and FOND4LTLf
with LAMA validates
\ExternalDirectValidCount{} (supported-set PAR-2
\ExternalDirectParTwoSeconds{} seconds) \cite{DeGiacomoFuggitti2021FOND4LTLf}.
The other \ExternalDirectUnsupportedCount{} inputs fall outside the reference's
declared input scope, so this is a task reference rather than a compiler
ablation.
