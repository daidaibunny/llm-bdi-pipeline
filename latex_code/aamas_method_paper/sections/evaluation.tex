\section{Evaluation}
\label{sec:evaluation}

The evaluation is organized around a fixed achievement-goal benchmark corpus,
not around learner-specific examples or implementation-local domains. The
current paper scope is positive conjunctive achievement goals. Temporal
extended goals are treated as a later goal-specification layer that can call the
same domain-level library, but they are not part of the core evaluation in this
paper.

All selected benchmark snapshots are materialized from one independent public
source: \texttt{AI-Planning/pddl-generators} at commit
\texttt{d5c22c9ab21ecaf90db82daf2a0537973c661009}. For each selected problem
class we track a local \texttt{domain.pddl}, twenty deterministic training
instances, ten deterministic goal-specification test instances, and a
\texttt{source.json} provenance record. The generated snapshots are committed
under \texttt{src/domains/<domain>/}; the external generator repository remains
an input, not a learner backend and not part of the synthesized library.

Table~\ref{tab:domain-families} follows the same style used by prior
generalized-planning work: support is stated over problem classes $Q_D$ with
recognized structural properties, rather than over arbitrary \pddl{} domains.
The first class follows the goal-independence and serialisability distinction
used by MOOSE \cite{Chen2026Moose}. The second class follows policy-sketch and
subgoal-width evaluations, where a compact sketch decomposes large problems into
bounded-width subproblems \cite{Drexler2022LearningSketches,
Drexler2024SubgoalSketches,Bonet2024GeneralPoliciesWidth}. The third class
follows feature-definable general-policy work, where Blocksworld variants expose
goal-dependent construction behavior that cannot be handled by arbitrary
singleton-goal ordering \cite{Frances2021LearningGeneralPolicies}.

The evaluation protocol has three checks. First, synthesis receives only the
training split and must produce a lifted \asl{} library whose plan heads are
declared \pddl{} predicate goals. Second, the generated library is executed on
the held-out goal-specification split without a run-time full-trace planner.
Third, the artifact validator checks that every reported row is reproducible
from the tracked manifest, uses the declared benchmark source, and does not
depend on obsolete implementation-local domains. Classical planners and
external generalized-planning systems may appear as offline baselines or
evidence providers, but they are not allowed to replace the run-time library.

\subsection{Reproducibility}

The reproducibility artifact is specified by
\texttt{paper\_artifacts/final\_paper\_manifest.json}. The manifest records the
benchmark source commit, required repository inputs, expected output policy,
resource limits for external learners, and the commands for generation,
validation, tests, and paper compilation. The achievement-goal registry under
\texttt{src/benchmark\_registry/achievement\_goals} is the single source for
matrix construction; experiment scripts consume registry rows instead of
embedding benchmark-specific paths.

Generated experiment outputs live under \texttt{tmp/paper-final-latest} and are
treated as rebuildable artifacts. Before paper submission, the final package
must be regenerated from the manifest, validated against the tracked benchmark
registry, and synchronized with the LaTeX result macros. This prevents stale
local runs, learner-specific benchmark copies, or deleted legacy domains from
entering the paper tables.
