\section{Experimental Evaluation}
\label{sec:evaluation}

\paragraph{Questions.}
We ask whether readable MOOSE evidence compiles into closed certified libraries
(RQ1), how much joint selection reduces their certified candidate spaces and
whether selection is symbol invariant (RQ2), whether supported DFA guards
compile without reordering DFA edges (RQ3), whether complete Jason traces pass
VAL and temporal validation (RQ4), and what library and execution costs result
(RQ5).

\paragraph{Corpus and protocol.}
The corpus contains all eight MOOSE companion classical domains with nonempty
official train/test splits \cite{Chen2026Moose}, four bounded-integer variants,
and four serialized-width families. Blocksworld Clear and On use the published
no-constants general-policy splits \cite{Frances2021LearningGeneralPolicies};
Tower uses the first quarter of a pinned public PDDL corpus and Depots the first
half of the D2L corpus \cite{Bonet2018Features}. These groups describe
evaluation properties and never route compilation.

MOOSE evidence used seed 0, three goal permutations, six workers, a 12-hour
timeout, and a 16-GiB guard. The paper reports 12 hours and 32 GB for synthesis,
five repetitions, and 30 minutes and 8 GB for test-time planning
\cite{Chen2026Moose}; our memory cap is a declared deviation. Temporal run
\texttt{\TEGExecutionRunID} uses clean commit
\texttt{\TEGExecutionCommit}, 12 workers, 30-minute Jason and VAL limits, and
a 64-MiB Java stack. It records SHA-256 hashes of every atomic JSON/ASL input.
The result generator rejects a changed benchmark hash, prediction hash, dirty
tracked revision, missing case, or mismatched library hash.

A nonempty execution succeeds only if Jason emits its explicit success marker,
neutral-goal VAL accepts the complete primitive PDDL trace, and replayed states
are accepted by both the gold and predicted DFAs. Failed prefixes are
diagnostics, never plans. The released benchmark contains short state-changing
witnesses: it evaluates the representation and execution chain, not solution of
the original PDDL achievement goal.

\paragraph{Gold and predicted conditions.}
The model is called once for each of \TranslationInputCount{} deduplicated
natural-language inputs. All \TranslationJSONValidCount{} predictions satisfy
the JSON/PDDL contract, and exact reachable-product exploration finds all
\TranslationEquivalentCount{} predicted DFAs language-equivalent to their gold
DFAs. The frozen run uses GPT-5.5, temperature 0, JSON-object output, a 60,000-
token ceiling, a 1,000-second request timeout, and at most three semantic
retries under prompt commit \texttt{8804f734}. Those predictions expand to
\TEGProblemCount{} bound query controllers.
We execute each predicted controller once and evaluate its resulting trace
against both automata. Thus gold-DFA and predicted-DFA acceptance are separate
oracles on the same trace, not two duplicated execution runs. Translation
failure would be recorded before controller execution and cannot be counted as
a BDI failure.

\paragraph{RQ1--RQ2: atomic libraries.}
Table~\ref{tab:domain-results} reports the exact hashed inputs used downstream.
All \BenchmarkDomainCount{} libraries contain readable MOOSE evidence, pass
PDDL support and module-closure checks, and record Clingo as the selector.
Across domains, Clingo selects
\AtomicSelectedBranchCount{} of \AtomicCandidateCount{} certified candidates,
a reduction of \AtomicRemovedBranchCount{} branches
(\AtomicReductionPercent\%), yielding \AtomicLibrarySizeKiB{} KiB of
ASL. This modest reduction is the measured compactness result: we claim
lexicographic optimality only inside each generated certified candidate space,
not a globally minimal AgentSpeak program. Predicate/action renaming,
type-compatible parameter permutation, irrelevant-fluent injection, and
counterexample-domain tests pass in the symbol-invariance suite. The atomic
generation batch did not record a clean source revision suitable for timing;
we therefore report its hashed library structure but omit synthesis and
compiler time from the formal runtime result.

\paragraph{RQ3--RQ5: temporal execution.}
Table~\ref{tab:profile-results} separates the five declared profiles. Every one
of the \TEGProblemCount{} bound queries compiles and succeeds in Jason; all
\TEGVALSuccessCount{} nonempty traces pass VAL, and both DFAs accept all
\TEGGoldDFAAcceptCount{} traces. No translation/schema error, unsupported DFA,
certificate rejection, Jason failure, timeout, VAL failure, or DFA rejection
occurs in this pinned matrix. The median trace has \TEGMedianActionCount{}
actions, median case runtime is \TEGMedianRuntimeSeconds{} seconds, and
12-worker wall time is \TEGWallRuntimeMinutes{} minutes. Complete coverage
supports the released formulas, bindings, and library hashes only; it is not an
action-strategy completeness theorem for arbitrary PDDL--\ltlf products.

\paragraph{Repetition boundary.}
The registered atomic protocol fixes five independent seeds, one internal MOOSE
worker, isolated outputs, and independent compilation and validation; it never
unions evidence or selects the best seed. Those repetitions are not yet a clean
result matrix. Table~\ref{tab:domain-results} therefore reports only the exact
seed-0, six-worker libraries consumed by the pinned temporal run and makes no
cross-seed or synthesis-time claim.

\paragraph{Independent semantic conformance.}
A separate versioned suite checks hand-labelled positive and negative traces
with both a direct recursive finite-trace evaluator and the real
\texttt{ltlf2dfa}/MONA DFA. Clean run
\texttt{\ConformanceRunID} passes
\ConformanceSemanticSuccessCount{}/\ConformanceSemanticCaseCount{} formula
cases and
\ConformanceZeroActionSuccessCount{}/\ConformanceZeroActionCaseCount{}
initial-state integration cases. A zero-action execution denotes the singleton
trace $\langle s_0\rangle$ and commits an empty action file. VAL 1.4 has no
zero-action syntax and rejects an empty file, so these two cases report VAL as
not applicable and action legality as vacuous after PDDL initial-state replay;
no noop is inserted. They remain outside the \TEGProblemCount{} nonempty-plan
denominator.
