\section{Problem Formulation and Foundations}
\label{sec:background}

\subsection{Planning Domains and Generalized Evidence}

We use typed STRIPS PDDL plus a bounded-integer resource fragment
\cite{FikesNilsson1971STRIPS,McDermott1998PDDL,FoxLong2003PDDL21}. A domain is
$D=\langle\mathcal{P},\mathcal{F},\mathcal{A},\mathcal{T}\rangle$ with predicate
schemas, integer-valued resource functions, action schemas, and a type
hierarchy; a state $s=(s_{\mathcal P},s_{\mathcal F})$ combines Boolean and
integer valuations under the standard add/delete successor. The numeric
component admits finite integer values, comparisons with constants, and
constant integer changes; arbitrary arithmetic and continuous change are
excluded. Actions are instantaneous and deterministic, and the state is fully
observable; conditional effects, durative actions, stochastic outcomes, and
partial observability are outside the model. A problem instance is
$I=\langle D,O_I,s_I^0,G_I\rangle$ with typed
objects, initial state, and PDDL achievement condition, and a
generalized-planning task supplies finite training and test instance sets
sharing $D$. MOOSE applies goal regression
\cite{Fikes1972GeneralizedRobotPlans,Alcazar2013Regression} to singleton goals
and lifts the plans into first-order condition-to-macro rules
\cite{Chen2026Moose}. We treat these rules as evidence: they demonstrate ways
to achieve observed singleton goals, but need not constitute a closed or
compact atomic module core.

\subsection{BDI Plan Libraries in AgentSpeak(L)}

A procedural BDI plan rule has a triggering event, an applicability context,
and a body \cite{MeneguzziDeSilva2015PlanningBDI}. In the \asl{} realization
evaluated here \cite{Rao1996AgentSpeak}, an achievement plan is
\[
+!p(\bar X):C\leftarrow b_1;\ldots;b_m,
\]
where $C$ is a conjunction of beliefs and comparisons and each $b_i$ is a PDDL
action or an internal achievement call $!q(\bar Y)$. A library is lifted when
plans contain variables rather than training objects, and internally closed
when every internal call resolves to a type-compatible selected module.

The evaluated target language is \asl{}, and Jason executes the selected lifted
rules \cite{Bordini2007Jason}. Extending the empirical claims to another BDI
language would require a semantics-preserving translation and an independent
execution study. Query controllers follow declarative goal patterns by
rechecking their intended state after subsidiary plans return
\cite{Hubner2006DeclarativeGoals}.

\subsection{Achievement and Temporal Goals}

An achievement goal requires a state condition to hold at completion
\cite{Rao1996AgentSpeak,Sardina2007PlanFailure}. For proposition alphabet
$AP_q$, we write such conditions as literals or conjunctions in the atemporal
language $\Phi_{\mathrm{ach}}$. A temporally extended goal (TEG) instead
constrains a finite execution trace \cite{Hindriks2009TemporalGoals}. We express
TEGs in linear temporal logic over finite traces (\ltlf{})
\cite{DeGiacomoVardi2013}. The declared language $\Phi_{\mathrm{syn}}$ extends
$\Phi_{\mathrm{ach}}$ with $F$ (eventually), $X$ (strong next), and $U$ (strong
until), with negation restricted to atoms. Thus a formula without a temporal
operator is an achievement goal, whereas a formula containing one is a TEG.
The full grammar and finite-trace semantics appear in the Technical Supplement,
Sec.~1. The evaluation family
$\Phi_{\mathrm{bench}}\subset\Phi_{\mathrm{syn}}$ contains instances of the
five predeclared profiles in Sec.~\ref{sec:evaluation}; membership in either
set is not an action-strategy guarantee.

Both goal classes use the same automata path. For
$\varphi_q\in\Phi_{\mathrm{ach}}$, the completion embedding is $F\varphi_q$; a
TEG is unchanged. Following automata-theoretic planning
\cite{DeGiacomoRubin2018AutomataFOND}, we construct the deterministic finite
automaton (DFA) for this execution formula
$\mathcal D_q=\langle Q_q^{\mathrm{dfa}},2^{AP_q},\delta_q,q_q^0,F_q\rangle$
with LTLf2DFA and MONA \cite{Fuggitti2020LTLf2DFA,Henriksen1995MONA}, and a
valuation map $\mathit{val}_q$ interprets a PDDL state over $AP_q$. For
automaton state $z$, let $d_q(z)$ be the shortest directed-path length to
$F_q$, or $\infty$ when no accepting state is reachable. A progress transition
$(q_s,\chi,q_t)$ satisfies $d_q(q_t)<d_q(q_s)<\infty$, and a supported guard
$\chi$ has normal form
\[
\chi=
\bigwedge_{G\in P_\chi^+}G\ \land\!
\bigwedge_{N\in P_\chi^-}\neg N,
\qquad
\mathcal O_\chi=\langle P_\chi^+,P_\chi^-\rangle,
\]
where the signed transition obligation $\mathcal O_\chi$ separates positive
achievements from atoms that must remain absent. We reserve four related terms.
An obligation is a condition the compiler must establish. A witness is finite
evidence for one obligation, such as a symbolic-progression derivation,
decreasing rank, or resource-discharge path. A certificate is the structured
record that identifies an artifact's applicable obligations and their witnesses;
for example, a transition certificate records portfolios, effect-summary
completeness, threats, and a safe order. Certification checks that record, and
certified denotes only an artifact that passes all required checks. The certified subset
$\Phi_{\mathrm{cert}}(D,\mathcal M_D)\subseteq\Phi_{\mathrm{syn}}$ contains the
validated bound formulas whose DFA progress transitions pass the checks in
Sec.~\ref{sec:temporal-composition}; soundness claims apply to this subset and
empirical claims to accepted members of $\Phi_{\mathrm{bench}}$.
A query-local monitor evaluates the DFA initially and after every successful
primitive PDDL action.

\subsection{Compilation Problems and Acceptance}

\paragraph{Atomic-core compilation.}
Given $D$, training instances
$\mathcal{I}_{\mathrm{train}}$, and normalized singleton-goal evidence $E$,
atomic compilation produces the certified atomic module core $\mathcal{M}_D$,
whose query-independent branches are triggered by singleton achievements.

\paragraph{Query compilation.}
For a validated bound query
$\widehat\tau_q=(\tau_q,\theta_q)$ whose binding maps each parameter to a
type-compatible object, query compilation takes $\mathcal D_q$ and $\mathcal M_D$
and produces $\mathcal Q_q$: top-level, DFA-dispatch, and transition-repair
plans. A conjunctive guard is compiled into a certified serialization of signed
atomic obligations, not into a trigger in $\mathcal M_D$. For appended queries
$q_1,\ldots,q_k$, the one maintained domain library is
\[
\mathcal{L}_D^{[0]}=\mathcal{M}_D,
\qquad
\mathcal{L}_D^{[k]}=
\mathcal{M}_D\cup\bigcup_{i=1}^{k}\mathcal{Q}_{q_i}.
\]
Thus $\mathcal M_D$ is the stable core of $\mathcal L_D^{[k]}$, not a
second persisted library, and appending $\mathcal Q_{q_i}$ neither reconstructs
nor reoptimizes the atomic core. Throughout, calligraphic symbols denote finite
sets or structured artifacts; $b$ denotes one candidate branch, $e$ one
normalized evidence rule, and $\chi$ one DFA transition guard. The Technical
Supplement, Sec.~1 gives the formal definitions, supported-fragment
assumptions, and the acceptance and rejection boundary used below.
