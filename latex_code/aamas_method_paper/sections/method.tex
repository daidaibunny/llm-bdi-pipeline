\section{Certified AgentSpeak Plan-Library Synthesis}
\label{sec:method}

\subsection{Method Overview}

This stage implements
$D,\mathcal{I}_{\mathrm{train}},E\longrightarrow\mathcal{L}_D$.
It compiles one reusable domain library $\mathcal{L}_D$ from a PDDL domain,
training split, and normalized generalized-planning evidence. Temporal queries
do not enter this optimization and never cause the atomic library to be
relearned. Algorithm~\ref{alg:atomic} gives the stage at a high level.

\begin{algorithm}[t]
\caption{Compile a Domain Atomic Library}
\label{alg:atomic}
\begin{algorithmic}[1]
\REQUIRE PDDL domain $D$, training split $\mathcal{I}$, provider artifact $A$
\ENSURE Lifted AgentSpeak library $\mathcal{L}_D$ or a rejection diagnostic
\STATE $E\leftarrow\textsc{NormalizeEvidence}(A,D,\mathcal{I})$
\STATE $T\leftarrow\textsc{ProducibleClosure}(E,D)$
\STATE $C_E\leftarrow\textsc{ValidateEvidenceMacros}(E,D)$
\STATE $C_D\leftarrow\textsc{GenerateSchemaCandidates}(T,D)$
\STATE $C\leftarrow\textsc{Certify}(C_E\cup C_D,D)$
\STATE $S\leftarrow\textsc{ClingoSelect}(C,E)$
\IF{$S$ violates coverage, closure, or a certificate obligation}
  \STATE \textbf{reject}
\ENDIF
\STATE \RETURN $\textsc{RenderAgentSpeak}(S)$
\end{algorithmic}
\end{algorithm}

\subsection{Evidence Normalization}

A \texttt{PolicyEvidenceProgram} is the provider-neutral intermediate
representation. It contains singleton-goal rules
\[
e=\langle \bar X,C_s,C_g,\bar a,\nu,\pi\rangle,
\]
where $\bar X$ are variables, $C_s$ is a partial state condition, $C_g$ is one
goal atom, $\bar a$ is a finite PDDL action sequence, $\nu$ contains numeric
guards or effects, and $\pi$ records provenance. For example, a MOOSE readable
rule for an object-location goal can contain a state condition, the goal
\texttt{at(P,L)}, and a \texttt{load; move; unload} macro. The adapter parses
and normalizes this rule but does not decide whether it is safe AgentSpeak code.

The implemented MOOSE adapter accepts only rules whose predicates, functions,
actions, arities, and constants are declared by $D$. Variables are
alpha-renamed so that two evidence rules cannot accidentally share local
bindings. Positive singleton predicates seed the module vocabulary. Negative
achievement evidence is rejected because the current provider contract does
not establish a negative-achievement implementation.

\subsection{Schema-Derived Candidate Space}

The compiler classifies each declared predicate structurally. A \emph{static
predicate} never appears in an action add or delete effect and may occur only in
contexts. A \emph{producible fluent} appears in a positive add effect and enters
the atomic candidate space even when it was never a training goal. This closure
also runs when the provider supplied only numeric-goal evidence; numeric
evidence must not suppress producible predicate modules. The same rule provides
schema-required helpers without assuming that MOOSE observed each helper as a
goal.

For each target $p(\bar X)$, candidate generation searches the fixed grammar in
Table~\ref{tab:grammar}. A \emph{producer} is an action whose add effects contain
$p$; a \emph{support} or \emph{bridge} is a producer for a missing positive
precondition of a later action. The grammar is domain independent but bounded.

\begin{table}[t]
\centering
\caption{Implemented schema-composition grammar.}
\label{tab:grammar}
\begin{tabular}{lp{0.57\columnwidth}}
\toprule
Candidate & Primitive-action shape \\
\midrule
Direct & target producer \\
Support & support producer; target producer \\
Bridge & support; bridge; target producer \\
Prefix & prefix; support; bridge; target producer \\
Cleanup & any preceding sequence; one certified release \\
\bottomrule
\end{tabular}
\end{table}

Schema-derived candidates therefore contain at most five primitive actions.
Validated evidence macros are not truncated to this bound. The bound defines
the hypothesis class searched by the compiler; it is neither a MOOSE parameter
nor a completeness claim for PDDL.

PDDL typing is used during unification and action binding. When Jason needs
runtime sort information, the compiler emits the reserved static context
predicate \texttt{obj\_tp(Object,Type)}. It never emits domain-specific
\texttt{type\_*} fluents, and \texttt{obj\_tp/2} is never an achievement call or
an exported action.

\subsection{Branch Certificates}

Every candidate carries a machine-checkable contract. The following
certificates are implemented.

\begin{table}[t]
\centering
\caption{Compiler certificates and the failures they exclude.}
\label{tab:certificates}
\begin{tabular}{lp{0.56\columnwidth}}
\toprule
Certificate & Excluded failure \\
\midrule
Binding & unbound action or subgoal variable \\
Execution & unsatisfied symbolic precondition \\
Achievement & branch returns without its trigger \\
Closure & selected call has no selected module \\
Progress & recursive branch can oscillate \\
Release & acquired resource remains occupied \\
Preparation & staged repair destroys feasibility \\
\bottomrule
\end{tabular}
\end{table}

\paragraph{Binding and schema execution.}
Every primitive-action or subgoal variable must be bound by the trigger, a
positive context literal, or \texttt{obj\_tp/2}. Negative contexts and
comparisons may filter existing bindings but may not introduce new variables.
Symbolic progression checks each action precondition against the context and
the effects of preceding actions. A finite action branch is accepted only when
its completion establishes its trigger atom. We record that atom as
$\mustadd$, and record conservative possible completion effects as $\mayadd$
and $\maydelete$.

\paragraph{Internal closure.}
If a body contains $!q(\bar Y)$, the selected library must contain a module for
$q$ whose trigger is type-compatible with the call. This closure obligation is
part of selection, rather than a post-processing check.

\paragraph{Recursive progress.}
A recursive branch is admitted only with a non-negative, well-founded ranking
feature. The implemented relational feature grammar contains (i) the number of
atoms of a dynamic relation and (ii) the number of relation atoms in an acyclic
cone anchored at a query argument. The recursive body must strictly decrease
the selected feature, no reachable selected branch may increase it, and a base
or producer branch must remain available. The feature is compile-time metadata;
it is not added as an AgentSpeak belief. This restriction follows the use of
relational features as progress witnesses in general-policy learning
\cite{Frances2021LearningGeneralPolicies} but claims only schema-certified
features from this fixed grammar.

\paragraph{Resource release.}
The compiler identifies a resource debt from causal effects rather than fluent
names. An acquire sequence deletes a key-only availability mode and adds a
key-plus-occupant debt mode; a release action must reverse that transition,
preserve the atomic target, and be symbolically executable. Required
disequalities are generated only when unification would violate a later action
precondition. The current search admits one release action and rejects
multi-step parking.

\paragraph{Existential preparation projection.}
Suppose one producer requires dynamic siblings $p_1,\ldots,p_k$. Requiring all
siblings before preparing any one of them can deadlock when a preparation must
temporarily change a single-valued state, such as an object's location. The
compiler builds a predicate dependency graph from positive producer
preconditions. It may defer an unrelated dynamic sibling from one preparation
context only if:
\begin{enumerate}
\item the relevant dependency graph is acyclic and traverses a
schema-inferred single-valued fluent;
\item every deferred sibling has exactly one producer schema;
\item that producer's static preconditions remain as feasibility witnesses;
\item nested variables are alpha-renamed away from outer variables; and
\item the branch calls the original target again, rechecking its complete
producer context before any primitive producer executes.
\end{enumerate}
For instance, calibration may be prepared before the final image waypoint is
re-established, while camera compatibility, calibration target, and visibility
facts remain in context. Cyclic or producer-ambiguous dependencies keep the
full connected context. Preparation priority is derived from the selected
module call graph only for branches carrying this certificate; unrelated
domains retain their original branch ordering.

\subsection{Joint Clingo Selection}

Validated evidence macros and certified schema candidates enter one answer-set
optimization problem implemented with Clingo \cite{Gebser2018Clingo}. Evidence
rules induce coverage obligations. A selected candidate may cover an evidence
obligation only through alpha-equivalent triggers and bodies, a weaker
conjunctive context with the same body, or a schema-producer refinement with
the same $\mustadd$, no additional $\maydelete$, and equivalent numeric and
resource contracts. Syntactic body-prefix similarity is not treated as semantic
equivalence.

The solver enforces evidence coverage, internal closure, producer availability,
and compatibility of recursive capabilities. Among feasible sets $S$, it uses
the lexicographic objective
\[
\min_S\bigl\langle-R(S),\ |S|,\ C(S),\ B(S)\bigr\rangle,
\]
where $R$ is the number of compatible certified recursive capabilities, $C$ is
the number of context literals, and $B$ is the number of body steps.

\section{DFA-Guided Composition of Temporally Extended Goals}
\label{sec:temporal-composition}

The temporal input is a validated tuple
$\langle q,g_q,\varphi_q,A_q,B_q\rangle$: a query identifier, a top-level
AgentSpeak goal, an \ltlf formula, declared atoms, and optional variable
bindings. The real \texttt{ltlf2dfa}/MONA output is parsed into DFA states and
guards. Accepting self-loops labelled \texttt{true} are ignored as plumbing;
every edge that reduces graph distance to acceptance receives a query-local
transition controller guarded by its runtime source-state belief. Valuation
cubes with the same source and target share only their common achievement
objective; the runtime monitor still evaluates every original complete cube.

One benchmark execution is an externally bound invocation. Before compiling
guards, the case assignment $B_q$ substitutes lifted parameters in the
query-local atoms; for example, $\mathit{on}(X,Y)$ under
$\{X\mapsto b_1,Y\mapsto b_4\}$ invokes $!\mathit{on}(b_1,b_4)$. This does not
ground the shared atomic library: its reusable trigger remains
$+!\mathit{on}(X,Y)$. Execution is rejected if the released invocation binding
differs from the sealed construction-audit assignment.

\begin{algorithm}[t]
\caption{Compile One Supported DFA Transition}
\label{alg:transition}
\begin{algorithmic}[1]
\REQUIRE Guard $B=P^+\land\neg P^-$, selected library $\mathcal{L}_D$
\ENSURE Query-local repair tree or a rejection diagnostic
\STATE $\Sigma\leftarrow\textsc{CompletionSummaries}(P^+,\mathcal{L}_D)$
\STATE $G\leftarrow\textsc{ThreatGraph}(P^+,P^-,\Sigma)$
\IF{$G$ is acyclic}
  \STATE $O\leftarrow\textsc{TopologicalOrder}(G)$
\ELSE
  \STATE $(O,S)\leftarrow\textsc{PreservingPortfolio}(G,\mathcal{L}_D)$
  \IF{no such $(O,S)$ exists}
    \STATE $(O,S)\leftarrow\textsc{WholeGuardAction}(B,D)$
    \IF{no such $(O,S)$ exists}
      \STATE \textbf{reject}
    \ENDIF
  \ENDIF
\ENDIF
\STATE \RETURN $\textsc{BalancedRepairTree}(O,P^-,S)$
\end{algorithmic}
\end{algorithm}

\subsection{Conditional Completion Effects}

A completion summary describes effects observable when an atomic module
successfully returns. It is computed as a relational fixed point over the
selected module call graph. Query arguments remain symbolic anchors, local
variables are alpha-normalized, and PDDL types restrict unification. Each effect
retains the positive, negative, equality, and disequality conditions of the
branch that produced it. Sequential composition records net completion
polarity: an atom deleted and restored inside one macro is not reported as a
completion delete.

For positive goals $G_i,G_j$, the threat graph contains edge
$G_j\rightarrow G_i$ when a feasible $\maydelete$ effect of a module for $G_j$
can unify with $G_i$. The edge requires $G_j$ to be repaired before $G_i$.
Schema-derived single-valued invariants also reject conjunctions that request
different values for the same key.

A negative guard $\neg N$ is not compiled as $!\neg N$. Every positive repair
must prove that no feasible $\mayadd$ effect can unify with $N$. If $N$ is
present, a signed negative leaf may call only a finite query-local branch whose
net effect has an exact $\mathsf{MustDelete}(N)$, establishes one positive
sibling, and preserves all other guard obligations. A negative-only edge remains
a context test and does not wait for an exogenous change. Free parameters of a
direct deleter are unified with compatible positive-sibling add effects before
range restriction, so both sides of a functional relocation are bound without
recognizing the fluent name.

\subsection{Cyclic Threats and Enforced Portfolios}

An acyclic threat graph is topologically serialized. For a cyclic universal
summary, the compiler may copy a certified subset of atomic branches under a
query-local alias. Finite action-only branches are admitted when exact symbolic
completion preserves earlier positive goals and all negative guards. A
recursive branch of the form $!p;!g$ is admitted only when it discharges the
explicit missing context $p$, the selected $p$ module has a complete summary,
and that summary preserves earlier obligations. Its recursive call is rewritten
to the query-local alias, preventing execution from escaping to an uncertified
sibling branch.

One additional ordering certificate applies to a functional, acyclic requested
graph over a single binary relation. The child/support orientation must come
from the recursive progress certificate, and every enforced branch must still
preserve earlier ranked achievements. The method assumes that this relation
remains acyclic in reachable states. Acyclic requested data alone is
insufficient; if branch-level noninterference cannot be proved, compilation
rejects.

\subsection{Numeric and Whole-Guard Certificates}

A positive integer equality may use a query-local primitive action whose PDDL
effect has a complete constant delta. Unit deltas are replayed only under a
strict directional guard, while a larger delta requires the exact predecessor
value. For a mixed Boolean/numeric guard, the compiler otherwise orders only
complete action-only net-effect summaries. If these summaries induce a cycle,
one primitive action is admitted only when symbolic execution proves all
Boolean additions, negative deletions, and numeric target equalities in the
same successor state. This bounded certificate does not claim arbitrary
arithmetic planning. A negated numeric equality remains observation-only when
no certified change-away action exists.

When a numeric producer has an unsatisfied positive precondition, a preparation
branch is admitted only if it strictly reduces the missing-precondition count,
preserves the remaining producer conditions, and leaves the target numeric
fluent unchanged. A constant-bounded numeric prerequisite is handled
analogously by a support action whose constant delta strictly reduces the
prerequisite deficit. These components form a lexicographic ranking; failure to
construct it causes rejection rather than an unproved recursive repair.

For a strong-Until source state, we intersect the non-progress self-loop cubes
and obtain source invariants. In the evaluated fragment each such state has one
positive progress literal. We inspect every primitive prefix of an action-only
macro. Before progress, no prefix may delete a positive source invariant or add
a negative one. The establishing step may consume an invariant because the
left operand of strong Until is required only at preceding trace positions. For
repeated unit numeric progress, the compiler derives a non-unification guard
separating the chosen action object from a protected object; an unavoidable
consuming step is restricted to the exact predecessor value. An observed
numeric-equality base case terminates recursive preparation.

Composition is capture avoiding. Before a support macro is inserted, its local
variables are alpha-renamed away from query variables and from variables owned
by the outer producer. Thus an accidental textual collision cannot force one
symbol to satisfy unrelated PDDL type or equality constraints.

\subsection{Balanced Declarative Repair Trees}

Given certified order $L_1,\ldots,L_n$, the tree visits the signed obligations
in that order. A positive leaf checks or achieves $L_i$; a negative leaf checks
$\neg N_i$ and otherwise calls only its certified establishment helper, if one
exists. An internal
node recursively executes the left and right halves of a contiguous range.
After one pass, a separate completion helper repeats the controller only while
the runtime monitor remains in its source state. Once a primitive action moves
the monitor away from that state, the helper returns and top-level dispatch
continues from the actual DFA state. This is necessary when one atomic macro
crosses several DFA edges before returning; requiring the immediately adjacent
target would replay an already completed transition. A rejecting successor has
no accepting shortcut and fails at top-level dispatch.

For $n=1$, the tree calls the same atomic module and checks source-state exit,
so its primitive-action behavior is the singleton case when one edge is crossed
and remains correct when a macro crosses several edges.

\subsection{Maintained Domain Library}

Each domain has one canonical JSON/AgentSpeak library pair. An accepted query
appends one top-level goal and its query-local helpers to this library. Atomic
modules are not regenerated per query. Query helper names may appear as
AgentSpeak goals but never as PDDL fluents or exported plan actions. Jason
executes the resulting program, the environment exports the complete primitive
action trace, and VAL independently checks that trace against the PDDL problem
\cite{Howey2004VAL}.

\section{Formal Guarantees}
\label{sec:guarantees}

The guarantees are conditional on the generated candidate space and supported
temporal fragment. They do not assert completeness for arbitrary PDDL domains,
AgentSpeak programs, or DFA strategies.

\paragraph{Atomic branch soundness.}
Let $c$ be an emitted candidate for trigger $p(\bar X)$.

\textbf{Proposition 1.} If the certified context of $c$ holds under a
type-consistent grounding and every called module satisfies its contract, then
$c$ is executable and $p(\bar X)$ holds when $c$ returns.

\textbf{Proof sketch.} The binding certificate grounds each action term.
Induction over body steps uses symbolic progression to establish the next
precondition. The achievement certificate establishes the trigger after the
final step. Recursive calls additionally rely on their well-founded ranking;
staged preparation rechecks the complete producer context before execution.
\hfill$\square$

This is a local applicable-branch property. It does not claim that candidate
generation finds an applicable branch in every reachable state.

\paragraph{Candidate-space optimality.}

\textbf{Proposition 2.} If Clingo returns $S$, every encoded evidence and
closure obligation is satisfied, and $S$ is lexicographically optimal within
the generated certified candidate space.

This follows from the hard constraints and optimization priorities in the
generated answer-set program. It does not imply optimality over all AgentSpeak
programs or completeness of candidate generation.

\paragraph{Supported-transition composition.}

\textbf{Proposition 3.} Let $B=P^+\land\neg P^-$ be an accepted transition
objective. If each required signed repair has an applicable certified branch,
each selected call terminates, and later repairs preserve earlier obligations,
then the controller either remains in the source state and retries or leaves it
through an actual DFA edge enabled by the observed valuation.

\textbf{Proof sketch.} Induct over the certified literal order. Each leaf makes
its signed literal. A negative establishment branch has an exact net delete.
The completion summary of every later leaf excludes a delete of an earlier
positive literal and an add of a forbidden negative atom. The deterministic
runtime monitor evaluates complete transition cubes after every primitive
action. Therefore source-state exit is evidence of a real enabled DFA edge,
including when it occurs inside an atomic macro; otherwise the controller
retries from the same source. \hfill$\square$

This is a soundness claim for the declared DFA fragment, not completeness of
action-strategy synthesis for arbitrary PDDL--LTLf products.

\paragraph{Balanced-tree structure.}

\textbf{Proposition 4.} For $n$ signed literals of which $e$ have a certified
repair helper, the generated tree contains $2n+e+2$ plans, has maximum trigger fan-out two and
nesting depth $O(\log n)$, visits $O(n)$ helpers per pass, and preserves the
certified order.

\textbf{Proof sketch.} There are $n$ satisfied-leaf plans, $e$ repair-leaf plans, $n-1$
internal nodes, one entry plan, and two completion plans. Midpoint splitting
gives logarithmic depth. Induction over contiguous ranges shows that in-order
traversal visits leaves in exactly the input order. \hfill$\square$
