\section{GP2PL Domain-Library Compilation}
\label{sec:method}

\subsection{Method Overview}

GP2PL first maps
$D,\mathcal{I}_{\mathrm{train}},E\longrightarrow\mathcal{L}_D$,
compiling one reusable domain library from a PDDL domain, training split, and
normalized generalized-planning evidence. Temporal queries do not enter this
optimization, so later queries reuse $\mathcal{L}_D$.
Algorithm~\ref{alg:atomic} summarizes the procedure.

\begin{algorithm}[htbp]
\caption{Compile a BDI Domain Plan Library}
\label{alg:atomic}
\begin{algorithmic}[1]
\REQUIRE PDDL domain $D$, training split $\mathcal{I}$, provider output $A$
\ENSURE Lifted BDI plan library $\mathcal{L}_D$ or a rejection diagnostic
\STATE $E\leftarrow\textsc{NormalizeEvidence}(A,D,\mathcal{I})$
\STATE $T\leftarrow\textsc{ProducibleClosure}(E,D)$
\STATE $C_E\leftarrow\textsc{ValidateEvidenceMacros}(E,D)$
\STATE $C_D\leftarrow\textsc{GenerateSchemaCandidates}(T,D)$
\STATE $C\leftarrow\textsc{Certify}(C_E\cup C_D,D)$
\STATE $S\leftarrow\textsc{ClingoSelect}(C,E)$
\IF{$S$ violates coverage, closure, or a certificate obligation}
  \STATE \textbf{reject}
\ENDIF
\STATE \RETURN $\textsc{RenderAgentSpeak}(S)$
\end{algorithmic}
\end{algorithm}

The certification and selection steps operate on trigger--context--body plan
rules before surface rendering. The final line is the language-specific step:
the evaluated realization maps $S$ to \asl{}, but the contribution is the
certified construction of the BDI plan library rather than a new AgentSpeak
semantics. The guarantees do not transfer to another BDI language without a
semantics-preserving translation and an independent execution study.

\subsection{Evidence Normalization}

An evidence provider is admissible when its output can be normalized into a
singleton-goal evidence program whose rules have the form
\[
e=\langle \bar X,C_s,C_g,\bar a,\nu,\pi\rangle,
\]
where $C_s$ is a partial state condition, $C_g$ is one goal atom, $\bar a$ is a
PDDL action macro, $\nu$ records numeric conditions and effects, and $\pi$
records provenance. For example, evidence for \texttt{at(P,L)} may contain a
\texttt{load; move; unload} macro. Admissibility requires every symbol and arity
to occur in $D$, capture-avoiding renaming of local variables, and a singleton
positive goal contract. MOOSE supplies the present instantiation
\cite{Chen2026Moose}. Negative
achievement evidence is rejected because the instantiated provider supplies no
action contract for achieving a negated literal.

\subsection{Schema-Derived Candidate Space}

Predicates are classified from effects: a static predicate appears in no add or
delete effect and remains context-only; every positive add-effect predicate is
a producible fluent and enters closure even if MOOSE never observed it as a
goal. Numeric-only evidence does not suppress this predicate closure.

For each target $p(\bar X)$, candidate generation searches the finite language
summarized in Table~\ref{tab:compiler-obligations}. A \emph{producer} is an
action whose add effects contain $p$. Backward STRIPS regression
\cite{Fikes1972GeneralizedRobotPlans,Alcazar2013Regression,Chen2026Moose}
replaces one
open positive precondition by a producer's preconditions and rejects a step
whose deletes contradict another open requirement. Producer preconditions are
first unified with compatible open requirements, after which only still-unbound
parameters receive fresh variables. Search is enabled only when the reachable
producer--precondition graph is acyclic.

Termination does not use an action-depth constant: a path cannot repeat an
alpha-normalized requirement/producer step or resource mode, and only the
shortest body is retained for an equivalent open-requirement and completion-
effect contract. Every candidate is then verified by symbolic progression.
Evidence macros are not truncated. Cyclic dependencies require provider
evidence or a separately certified recursive module. This defines a restricted candidate
language rather than a complete PDDL planner. Types constrain unification; a
\asl{} realization may expose static sort membership in contexts, but never
as an achievement call or exported action.

\paragraph{Binding, execution, and closure.}
Action and subgoal variables must be bound by the trigger, positive context, or
certified static sort membership; negative contexts and comparisons only
filter. Symbolic progression proves each precondition and final trigger
achievement, recording $\mustadd$, $\mayadd$, and $\maydelete$. Every selected
call $!q(\bar Y)$ must have a type-compatible selected module; closure is a
solver obligation.

\paragraph{Recursive progress.}
A same-predicate recursive branch requires a non-negative relation count or
anchored acyclic-cone count that strictly decreases and has a base/producer exit.
Threats are checked over the preparation graph selected by Clingo, not over
unselected candidates. A branch may add the ranked relation outside the
protected cone only when its trigger preserves the same anchor and a guard proves
that the new anchor differs from the protected one. This certificate feature
never becomes an agent belief. The design follows relational progress witnesses
in general-policy work \cite{Frances2021LearningGeneralPolicies} but claims only
schema-certified features from the generated candidate language.

\paragraph{Resource release.}
Effects, not predicate names, identify temporary occupation of a reusable
resource. Suppose an acquisition action deletes an availability fact
$\mathit{free}(R)$ and adds an occupancy relation $\mathit{held}(R,O)$. These
names are illustrative: the compiler infers the shared resource argument $R$
and occupant argument $O$ from the effect structure and obtains the same result
after consistent symbol renaming. A discharge body is accepted only if symbolic
execution removes the current occupancy mode, restores the corresponding
availability fact, leaves the atomic target true, and does not recreate an
earlier normalized mode. This yields a finite restoration path without assuming
that release is one action. Disequalities arise only when unifying two roles
would make a PDDL precondition false.

\paragraph{Existential preparation projection.}
Preparing one of several dynamic preconditions may temporarily change another,
such as an object's location. Projection may defer an unrelated sibling only
through a schema-inferred single-valued fluent when every deferred sibling has
one producer, all static feasibility witnesses remain, local variables are
alpha-renamed, and the original target is retried before primitive production.
Cross-predicate preparation candidates are optional. Clingo selects an acyclic
subset and seals every selected edge with natural-number caller and callee ranks
satisfying $r_{caller}>r_{callee}$. Same-predicate recursion instead requires the
relational certificate above.

\subsection{Joint Clingo Selection}

Validated evidence macros and certified schema candidates enter one answer-set
optimization problem solved with Clingo \cite{Gebser2018Clingo}. Evidence
rules induce coverage obligations. A selected candidate may cover an evidence
obligation only through alpha-equivalent triggers and bodies, a weaker
conjunctive context with the same body, or a schema-producer refinement with
the same $\mustadd$, no additional $\maydelete$, and equivalent numeric and
resource contracts. Syntactic body-prefix similarity is not treated as semantic
equivalence.

The solver enforces evidence coverage, internal closure, producer availability,
acyclic ranked cross-predicate preparation, and compatibility of relation
effects with recursive capabilities. Among feasible sets $S$, it uses the
lexicographic objective
\[
\min_S\bigl\langle-R_{rel}(S),\ -R_{prep}(S),\ |S|,\ C(S),\ B(S)\bigr\rangle,
\]
where $R_{rel}$ counts compatible relation-ranked recursive capabilities,
$R_{prep}$ counts compatible acyclic precondition-discharge capabilities, $C$
is the number of context literals, and $B$ is the number of body steps. This is
global optimality only inside the generated and certified candidate space.
If $\mathcal{C}^{\checkmark}_D$ denotes candidates whose certificates hold and
$\Omega_E$ the evidence and closure obligations, the construction is
equivalently
\[
\operatorname*{lexmin}_{S\subseteq\mathcal{C}^{\checkmark}_D}
\operatorname{cost}(S)
\quad\text{subject to}\quad S\models\Omega_E.
\]
This constrained selection, rather than the surface AgentSpeak syntax, defines
the reusable domain-library result.

The compiler rejects an untyped predicate with several producers when their
safe use requires different nested branch portfolios. For example, if one
\texttt{at/2} producer preserves a ranked relation and another may increase it,
static role names alone do not prove that their object sets are disjoint. The
compiler neither recognizes those names nor assumes disjointness.

\section{DFA-Guided Composition of Temporally Extended Goals}
\label{sec:temporal-composition}

\paragraph{Validated parametric input.}
GP2PL does not prescribe a controlled surface grammar for user requests. In the
evaluated front end, a source utterance is accompanied by the public PDDL symbol
catalogue, declared typed parameters, and binding constraints. Translation
produces a structured specification
$\tau_q=\langle id_q,\varphi_q,A_q,P_q,C_q\rangle$, where $\varphi_q$ is an
\ltlf{} formula over proposition symbols, $A_q$ maps each symbol to a lifted
PDDL predicate or integer equality, $P_q$ declares the parameters, and $C_q$
constrains their bindings. The serialized artifact has exactly eight fields:
three fixed metadata fields for schema version, sample identity, and temporal
logic; the formula and atom table; the parameter declarations and constraints;
and a support-status field. Before automaton construction, the validator rejects
rather than repairs an artifact unless every formula symbol has exactly one
atom-table entry, no entry is unused, every symbol belongs to the PDDL catalogue,
and all arities, types, numeric values, and temporal operators satisfy the
declared fragment. The controlled utterances in our released benchmark define
the evaluated input distribution, not a user-facing query syntax. The frozen
prompt and complete JSON schema appear in the Technical Supplement, Sec.~4.2.

A temporal query is represented by
$\langle q,g_q,\varphi_q,A_q,B_q\rangle$: a query identifier, top-level goal,
\ltlf{} formula, atom table, and parameter binding. The
\texttt{ltlf2dfa}/MONA translation supplies the automaton states and guarded
transitions \cite{Fuggitti2020LTLf2DFA,Henriksen1995MONA}. An accepting
\texttt{true} self-loop requires no achievement controller. Each edge that
reduces graph distance to acceptance receives a controller guarded by its
source state. Valuation cubes with the same source and target may share their
common achievement objective, while the monitor continues to evaluate each
complete cube.

Each execution externally binds $B_q$: $\mathit{on}(X,Y)$ under
$X{=}b_1,Y{=}b_4$ calls $!\mathit{on}(b_1,b_4)$ without grounding the shared
$+!\mathit{on}(X,Y)$ module. A binding inconsistent with the validated query
specification is rejected.
For guard $P^+\land\neg P^-$, compilation computes conditional completion
summaries and a threat graph, building on BDI goal-interference analyses that
distinguish definite from potential plan effects
\cite{Thangarajah2003GoalInterference}. An acyclic graph is topologically
ordered; a cyclic graph requires either a preserving branch portfolio or an
action that establishes the entire guard. The transition is rejected if neither
can be certified.
When one action establishes the complete guard, its repair plan is callable from
every positive literal it establishes and carries the full certified anchor
tuple. Thus an already-true first literal cannot hide the action from an unmet
sibling or discard a query binding.
The certified order is rendered as a balanced repair tree.

\subsection{Conditional Completion Effects}

A relational fixed point over selected calls summarizes effects observable on
module return. Query arguments remain anchors; locals are alpha-normalized and
typed; effects retain branch conditions. Sequential composition records net
polarity, so an internally deleted-and-restored atom is not a completion delete.

For positive goals $G_i,G_j$, the threat graph contains edge
$G_j\rightarrow G_i$ when a feasible $\maydelete$ effect of a module for $G_j$
can unify with $G_i$. The edge requires $G_j$ to be repaired before $G_i$.
Single-valued invariants reject conjunctions requesting two values for one key.

A negative guard $\neg N$ never becomes $!\neg N$. Positive repairs must exclude
feasible $\mayadd(N)$. If $N$ holds, a signed leaf may call only a finite branch
with exact $\mathsf{MustDelete}(N)$ that establishes a positive sibling and
preserves all obligations. A negative-only edge observes absence. Direct-
deleter parameters bind through compatible sibling add effects before range
restriction, without recognizing fluent names.

\subsection{Cyclic Threats and Enforced Portfolios}

For a cycle, a query-local alias exposes only branches whose exact completion
preserves prior positives and negatives. Portfolios are certified per ordered
literal occurrence: two occurrences of $p$ may need different aliases when
their protected prefixes differ, while identical branch sets may share one.
Contexts remain alternative applicability cases, not simultaneous obligations.
A recursive $!p;!g$ branch also requires a complete preserving summary for $p$
and rewrites $!g$ to the alias. Certificates record source branches by literal
index and claim recursive closure only when one was selected. A functional,
acyclic binary request graph may follow its certified progress orientation,
assuming reachable acyclicity, but every branch must still preserve earlier
achievements; otherwise compilation rejects.

\subsection{Numeric and Whole-Guard Certificates}

A positive integer equality may call a complete constant-delta action: unit
deltas require strict direction, larger deltas the exact predecessor. Mixed
guards use complete action-only net effects. For example, a body for
$\mathit{at}(P,L)\land \mathit{fuel}(V)=3$ must symbolically establish
$\mathit{at}(P,L)$ and the exact integer value 3 in its final state while
preserving every other signed guard obligation. A cyclic guard admits one action
only if the same proof covers the whole successor state. Negated equality is
observation-only without a certified change-away action; arbitrary arithmetic
and independent ``solve then hope'' composition are not claimed.

Numeric preparation must strictly reduce missing preconditions or a constant-
bounded deficit, preserve other conditions, and leave the target fluent
unchanged. These components form a lexicographic ranking; no proof means reject.

For strong Until, intersected non-progress self-loops define source invariants.
Before the single progress literal, no action prefix may delete a positive
invariant or add a negative one; the establishing step may consume the left
operand. Repeated unit progress separates action and protected objects by a
derived non-unification guard, and unavoidable consumption requires the exact
predecessor. An observed target equality is the base case.

Inserted support locals are alpha-renamed away from query and producer
variables, preventing textual collisions from imposing unrelated constraints.

\subsection{Balanced Declarative Repair Trees}

Given order $L_1,\ldots,L_n$, positive leaves check or achieve $L_i$, negative
leaves check absence or call one certified repair plan, and internal nodes execute
contiguous left/right halves. Completion repeats only while the monitor remains
at the source; after any macro crosses one or more DFA edges, top-level dispatch
resumes from the observed state. Rejecting successors fail. For $n=1$, one leaf
calls the same atomic module and checks source exit. The repeated state check is
an instance of the declarative goal plan pattern
\cite{Hubner2006DeclarativeGoals}; the balanced dispatch structure is GP2PL's
controller realization.

\subsection{Persistent Domain Library}

Compilation yields one persistent library $\mathcal{L}_D$ and one \asl{}
realization per domain. Each query extends $\mathcal{L}_D$ with one top-level
goal and its local controllers without reconstructing the atomic modules;
controller goals are neither PDDL fluents nor exported actions. Jason records
the complete primitive trace, which VAL checks against the PDDL problem
\cite{Howey2004VAL}.

If the initial valuation is accepting, the committed plan is empty and the
trace is $\langle s_0\rangle$, not an empty trace or inserted noop; a noop could
change strong-Next or Until truth under finite-trace semantics
\cite{DeGiacomoVardi2013}.

\section{Formal Guarantees}
\label{sec:guarantees}

The guarantees are conditional on the generated candidate space and supported
temporal fragment. They do not assert completeness for arbitrary PDDL domains,
AgentSpeak programs, or DFA strategies.

\paragraph{Atomic branch soundness.}
\textbf{Proposition 1.} If an emitted $p(\bar X)$ branch's context holds under a
type-consistent grounding and called modules satisfy their contracts, the branch
is executable and returns with $p(\bar X)$. Binding grounds each term; induction
over symbolic progression establishes preconditions and final achievement;
recursion uses its ranking and preparation rechecks the producer context.
\hfill$\square$ This is local soundness, not reachable-state coverage.

\paragraph{Candidate-space optimality.}

\textbf{Proposition 2.} If Clingo returns $S$, every encoded evidence and
closure obligation is satisfied, and $S$ is lexicographically optimal within
the generated certified candidate space.

This follows from answer-set constraints and priorities, not from completeness
of candidate generation or optimality over all AgentSpeak programs.

\paragraph{Supported-transition composition.}

\textbf{Proposition 3.} Let $B=P^+\land\neg P^-$ be an accepted transition
objective. If each required signed repair has an applicable certified branch,
each selected call terminates, and later repairs preserve earlier obligations,
then the controller either remains in the source state and retries or leaves it
through an actual DFA edge enabled by the observed valuation.

\textbf{Proof sketch.} Induct over the certified order: each leaf establishes
its signed literal; later summaries exclude deleting prior positives or adding
forbidden atoms. The monitor evaluates full cubes after every action, so source
exit denotes an edge of the constructed DFA, including when a module traverses
that edge before returning; otherwise the controller retries.
\hfill$\square$ This does not imply complete action-strategy synthesis.

\paragraph{Balanced-tree structure.}

\textbf{Proposition 4.} For $n$ signed literals and $e$ repair plans, the tree
has $2n+e+2$ plans, fan-out two, depth $O(\log n)$, and $O(n)$ visits per pass,
while preserving order. The count follows from $n$ satisfied leaves, $e$ repair
leaves, $n-1$ internal nodes, one entry, and two completion plans; midpoint
splitting and range induction establish depth and order. \hfill$\square$

\paragraph{Monitor trace fidelity.}

\textbf{Proposition 5.} Let
$s_0,a_1,s_1,\ldots,a_k,s_k$ be the PDDL execution observed by the monitor, and
let $q_0$ be the DFA state after evaluating $s_0$. After every successful
primitive action $a_i$, the monitor belief equals
$q_i=\delta(q_{i-1},\mathit{val}(s_i))$. Hence leaving a controller source state
is exactly an exit in the deterministic automaton run over the observed finite
trace.

The result follows by induction on primitive actions: the environment first
applies the PDDL successor and then evaluates the deterministic transition
function on the resulting valuation. Controller plans cannot write monitor
beliefs. \hfill$\square$

\paragraph{Initial-state identity.}

\textbf{Proposition 6.} If the initial valuation is accepting, successful
execution commits no primitive action and denotes the one-state finite trace
$\langle s_0\rangle$. It is therefore equivalent to evaluating the query on the
initial state and is not equivalent to inserting an arbitrary noop.

This is immediate from finite-trace semantics and Proposition~5: the initial
valuation is consumed before action execution, while a noop would introduce a
second observed position and could change strong-Next or Until truth.
\hfill$\square$

Complete proofs, including the binding, symbolic-progression,
relational-ranking, resource-discharge, and tree-order lemmas used by these
propositions, appear in the Technical Supplement, Sec.~2.
