\section{Related Work and Positioning}
\label{sec:related-work}

\paragraph{Generalized planning.}
Generalized planning seeks compact policies or programs that solve families of
planning instances
\cite{Jimenez2019ReviewGeneralizedPlanning,Srivastava2011DirectedSearch,
Bonet2018Features,Frances2021LearningGeneralPolicies,
Bonet2024GeneralPoliciesWidth}, via policy sketches and answer-set optimization
\cite{Drexler2022LearningSketches,Drexler2024SubgoalSketches}, lifted decision
lists \cite{Yang2022PG3}, or terminating programs
\cite{SegoviaAguas2022BFGPpp}; MOOSE learns lifted condition-to-macro rules by
singleton-goal regression \cite{Chen2026Moose}. Recent BDI integrations invoke
a classical planner at run time \cite{Xu2024OperationalSemantics} or use
generalized planning for means--ends reasoning
\cite{Meneguzzi2025GeneralisedBDIPlanning}. GP2PL instead addresses the
representation-compilation problem: transforming singleton-goal evidence and
action-schema-derived candidates into a query-independent atomic module core
$\mathcal M_D$ that later queries extend.

\paragraph{Modular policies and BDI plan libraries.}
Policy-reuse languages let one generalized policy invoke another with
parameters \cite{Bonet2024PolicyReuse}, motivating internal calls such as
\texttt{!clear(Y)}. Procedural BDI languages organize behavior as
event-triggered plans selected from a context-dependent library
\cite{MeneguzziDeSilva2015PlanningBDI,DeSilva2020BDIArchitectures,
Rao1996AgentSpeak,Bordini2007Jason}, with declarative-goal rechecking
\cite{Hubner2006DeclarativeGoals}, plan-failure handling
\cite{Sardina2007PlanFailure}, and effect-summary interference analysis
\cite{Thangarajah2003GoalInterference}. GP2PL adopts these execution
principles and derives certified modules, completion summaries, and
declarative completion controllers from planning evidence and PDDL schemas.

\paragraph{Finite-trace temporal control.}
\ltlf{} admits automata-based reasoning \cite{DeGiacomoVardi2013}, which we
instantiate with the MONA-backed LTLf2DFA construction
\cite{Fuggitti2020LTLf2DFA,Henriksen1995MONA}. Full temporal synthesis solves
the domain--automaton product \cite{DeGiacomoRubin2018AutomataFOND}; GP2PL
instead composes certified repairs over an existing BDI library and makes no
complete product-synthesis claim. Temporally extended goals also have explicit
agent-language semantics \cite{Hindriks2009TemporalGoals}; we retain standard
plan execution and add a query-local monitor. Following the fixed-planner
principle of \cite{Bonassi2023Plan4Past}, FOND4LTLf
\cite{DeGiacomoFuggitti2021FOND4LTLf} is a task-level reference on its accepted
subset, not a compiler ablation.

\paragraph{Natural-language temporal specifications.}
Prior systems map natural-language instructions to temporal logic through
pattern libraries \cite{FuggittiChakraborti2023NL2LTL}, lifted
language-to-logic translation \cite{Chen2023NL2TL,Guo2025CaStL}, or modular
grounding \cite{Liu2023Lang2LTL}; prompt-based parsers remain sensitive to
input perturbations
\cite{BeurerKellner2023Prompting,Zhuo2023RobustSemanticParsing}. Translation is
an evaluated input condition here, not a contribution: we validate typed,
externally bound parametric \ltlf{} by exact finite-trace language equivalence
before GP2PL compilation, with the protocol and frozen prompt in the Technical
Supplement, Sec.~4.

\paragraph{Position of this work.}
Prior work supplies generalized policies, answer-set optimization, AgentSpeak
execution semantics, and \ltlf{} automata as separate artifacts. GP2PL
formulates their connection as a certificate-carrying compilation problem that
establishes schema-level obligations before emitting one maintained executable
library, echoing proof-carrying code \cite{Necula1997ProofCarryingCode} with
narrower symbolic planning obligations. Goal regression, answer-set solving,
AgentSpeak semantics, and \ltlf-to-DFA translation remain prior components.
