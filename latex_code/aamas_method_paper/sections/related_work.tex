\section{Related Work and Positioning}
\label{sec:related-work}

\paragraph{Generalized planning.}
Generalized planning seeks compact policies or programs that solve families of
planning instances
\cite{Jimenez2019ReviewGeneralizedPlanning,Srivastava2011DirectedSearch,
Bonet2018Features,Frances2021LearningGeneralPolicies,
Bonet2024GeneralPoliciesWidth}, via policy sketches and answer-set optimization
\cite{Drexler2022LearningSketches,Drexler2024SubgoalSketches}, lifted decision
lists \cite{Yang2022PG3}, or terminating programs
\cite{SegoviaAguas2022BFGPpp}; MOOSE learns lifted condition-to-macro rules by
singleton-goal regression \cite{Chen2026Moose}. Program-generating alternatives
include LMPlan's PDDL-sound Python policies \cite{Chen2025LMPlan} and GenePlan's
evolved domain-specific Python planners \cite{Murray2026GenePlan}. Recent BDI
integrations invoke a classical planner at run time
\cite{Xu2024OperationalSemantics} or use generalized planning for means--ends
reasoning \cite{Meneguzzi2025GeneralisedBDIPlanning}. Rather than synthesize a
standalone executor, GP2PL compiles evidence and action schemas into a closed
atomic module core $\mathcal M_D$ that later queries extend.

\paragraph{Modular policies and BDI plan libraries.}
Policy-reuse languages permit one generalized policy to invoke another with
parameters \cite{Bonet2024PolicyReuse}, motivating internal calls. BDI work also
studies plan-failure handling
\cite{Sardina2007PlanFailure} and effect-summary interference
\cite{Thangarajah2003GoalInterference}. Building on these mechanisms, GP2PL
derives certified modules, completion summaries, and declarative completion
controllers from planning evidence and PDDL schemas.

\paragraph{Finite-trace temporal control.}
\ltlf{} admits automata-based reasoning \cite{DeGiacomoVardi2013}, which GP2PL
instantiates with the MONA-backed LTLf2DFA construction
\cite{Fuggitti2020LTLf2DFA,Henriksen1995MONA}. Full temporal synthesis solves
the domain--automaton product \cite{DeGiacomoRubin2018AutomataFOND}. FOND4LTLf
compiles \ltlf{} goals into standard FOND planning
\cite{DeGiacomoFuggitti2021FOND4LTLf}; Plan4Past compiles pure-past goals into
classical planning \cite{Bonassi2023Plan4Past}. TIDE instead
searches automaton traces and backtracks over classical reach--avoid subproblems
\cite{Suprun2026TIDE}. GP2PL performs neither product nor trace search: it
compiles certified repairs over the domain's compiled BDI library and makes no
complete strategy-synthesis claim. Temporally extended goals have explicit
agent-language semantics \cite{Hindriks2009TemporalGoals}; GP2PL retains
plan execution and adds a query-local monitor.

\paragraph{Natural-language query specifications.}
Prior systems map natural-language instructions to temporal logic through
pattern libraries \cite{FuggittiChakraborti2023NL2LTL}, lifted
language-to-logic translation \cite{Chen2023NL2TL,Guo2025CaStL}, or modular
grounding \cite{Liu2023Lang2LTL}. Translation is
part of GP2PL's input contribution: unlike unrestricted semantic parsing, our
contract preserves typed, externally bound parameters in a fixed structured
representation for achievement goals and TEGs, validates PDDL vocabulary and
types, and evaluates temporal predictions against gold formulas by exact
finite-trace language equivalence.
