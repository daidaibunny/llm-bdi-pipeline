\section{Related Work and Positioning}
\label{sec:related-work}

Policy sketches express domain-general subgoal structure as rules over
qualitative feature changes \cite{Drexler2024SubgoalSketches}. The
learner-sketches system learns such sketches automatically through an answer-set
formulation \cite{Drexler2022LearningSketches}. This line provides the closest
theoretical basis for our method because it studies subgoal serialization and
bounded-width decompositions. The difference is the target artifact. A raw
sketch normally guides a planner that fills in subproblems; our compiler instead
turns selected sketch evidence into executable lifted \asl{} rules.

Generalized-policy learning studies policies that solve many instances of a
domain using learned features and abstract transitions
\cite{Bonet2018Features,Frances2021LearningGeneralPolicies,
Bonet2024GeneralPoliciesWidth}. These methods are attractive because they
avoid per-problem trace replay. However, a flat policy is not already an agent
plan library: it does not expose predicate-goal modules, reusable subgoal calls,
or the top-level \texttt{+!g} composition structure expected by
Belief-Desire-Intention agents. Policy reuse moves closer to our target by
allowing policies to call other policies \cite{Bonet2024PolicyReuse}. We adopt
that modular intuition, but make the interface concrete: Layer B produces
predicate achievement modules, and Layer C calls those modules under
goal-conditioned contexts.

MOOSE learns first-order condition-to-action rules by goal regression from
training plans \cite{Chen2026Moose}. It is a strong comparison point because it
also seeks domain-level generalized plans from training problems. Our local
reproduction indicates that its assumptions are suitable for some domains, but
less suitable as the final mechanism for strongly goal-dependent Blocksworld
variants. We therefore use MOOSE as a baseline and source of contrast, not as
the compiler target. Our output must be a lifted \asl{} library whose heads are
declared predicates and whose body steps are declared primitive actions or
predicate subgoals.

Lifted decision-list policies and program-synthesis approaches provide useful
alternative hypothesis classes. PG3 searches over goal-conditioned lifted
decision lists and uses policy-guided planning as the score
\cite{Yang2022PG3}. BFGP++ searches over terminating C++-style generalized
programs with loops \cite{SegoviaAguas2022BFGPpp}. These representations are
more directly executable than raw sketches, but they do not target the
Belief-Desire-Intention plan-library interface: predicate achievement modules,
subgoal calls, and a reusable \texttt{+!g} composer.

Answer-set programming is used as an optimization and consistency mechanism,
following the style of sketch learning with Clingo
\cite{Gebser2018Clingo,Drexler2022LearningSketches}. The answer set does not
invent arbitrary program text. It selects among candidates whose symbols have
already been checked against the \pddl{} domain, the external feature-binding
contract, and the compiler's variable-safety rules.

External code reuse is intentionally conservative. The sketch learner is the
primary consumed backend: its feature-rule policies have a verified parser,
feature-binding audit, and conservative \asl{} compiler path. The h-policy and
d2l artifacts are pinned and admitted only through verified safe dialect
adapters. Unsupported feature constructs remain diagnostics rather than
executable plan-library evidence. This prevents a common failure mode in which a
qualitative feature policy is treated as if it were already an executable agent
program.
